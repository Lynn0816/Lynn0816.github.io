使用Hexo與Github Pages搭建個人部落格

發表於 2022-04-25

前言

Github pages是Github提供的網頁代管服務，透過文件來維護更新，不使用資料庫來進行管理。除了Hexo可以作為靜態網站產生器(Static Site Generator)之外，Hugo和JekyII也可以作為選擇。

靜態網站優點在於

  • 速度快
  • 不需要資料庫
  • 安全性高

Hexo是基於Node.js的框架，使用對於網頁開發者很熟悉的程式語言JavaScript，支援Markdown語法建構部落格文章，只需要一個指令便可以佈署到Github，截至目前有362個主題可供選用，吸引我的一大原因也是因為Hexo的創建者是台灣人，對於東方語言的支持友善，擁有廣大的台灣與中國使用者。

前置作業

確認環境已安裝下列工具

  • Git
  • Node.js︰版本需不低於8.10，建議使用10.0及以上版本

Hexo安裝

開啟cmd透過npm來安裝Hexo

測試Hexo是否安裝成功

Hexo初始化

接著是初始化Hexo。
首先選擇好路徑，創立一個資料夾<folder>作為將來的部落格目錄，所有有關部落格的文件、資料都將儲存於此路徑底下。

  • 請重新命名<folder>，可以使用<username>.github.io作為資料夾名稱方便作業。

初始化完成，瀏覽新創立的資料夾，該目錄底下會有一些自動產生的檔案。

  • themes 是存放Hexo主題的資料夾
  • source 利用Markdown撰寫的部落格文章(.md)將存放在這裡
  • _config.yml 配置大部份的部落格設定的重要文檔

在本地端啟動Hexo server來預覽網頁。

  • 瀏覽器開啟http://localhost:4000/
  • Hexo初始主題配置為landscape
  • 下篇文章會教學如何更換主題與設定(Hexo主題大集合)

將Hexo佈署到Github

  • 開啟你的Github帳號，右上角+->New repository
  • Repository name建議命名為<username>.github.io

回到部落格目錄底下安裝插件hexo-deployer-git

修改_config.yml檔案，repo是剛建立的Repository網址。

最後要輸入三個指令，Clean, Generate and Deploy(清除緩存、生成靜態網頁、佈署)，就能成功部屬並更新至Github。

瀏覽器開啟<username>.github.io，就可以看到簡單的公開網頁供所有使用者瀏覽！
